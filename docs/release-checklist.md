# Public release checklist

Before publishing:

- [ ] Remove all production secrets.
- [ ] Replace production domain names if needed.
- [ ] Confirm no real user data is included.
- [ ] Confirm the repo is described as a PoC, not a complete package.
- [ ] Add screenshots or a short demo video if available.
- [ ] Add a clear note that some host-app dependencies are not included.
- [ ] Add a link to the portfolio project page.
- [ ] Create the first GitHub release as `v0.1.0-poc`.
